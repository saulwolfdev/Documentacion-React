# Componentes

Los componentes en React son bloques de construcción reutilizables que permiten estructurar la interfaz de usuario de una aplicación de manera modular. 
Cada componente puede contener su propio estado y lógica, y puede ser compuesto por otros componentes para construir interfaces más complejas.

### Definición de Componente:

En React un componente puede ser una función o una clase de JavaScript que devuelve elementos de React (usualmente, representados mediante [JSX](/docs/conceptos-de-react-09-jsx--docs)) que 
describen cómo se debe renderizar la interfaz de usuario.

## Componentes Funcionales:

### Sin variable:

**Ventajas:** Más conciso y a menudo se prefiere para componentes simples.

**Uso común:** Para componentes pequeños y funcionales que no necesitan ser referenciados en múltiples lugares.

<CodeBlock
  codeString={`
  function Saludo(props) {
    return <p>Hola, {props.nombre}!</p>;
  }
  `} 
  language='javascript'
/>

### Con variable:

**Ventajas:** Puede ser útil para componentes más grandes o cuando se necesita reutilizar el componente en varios lugares.

**Uso común:** Para componentes más complejos o cuando se desea una referencia fácilmente reutilizable.

<CodeBlock
  codeString={`
  const Saludo = function(props) {
    return <p>Hola, {props.nombre}!</p>;
  };
  `} 
  language='javascript'
/>

<DynamicSandpack
dynamicFiles={{    
"/App.js": 
`import React from 'react';

  return (
    <div>
      <Saludo nombre="Usuario" />
    </div>
  );
}

export default App;`, 

"/Saludo.js":
`import React from 'react';

const Saludo = (props)=> {
    return <p>Hola, {props.nombre}!</p>;
};

export default Saludo;`,
  }}
  
  dynamicVisibleFiles={["/App.js", "/Saludo.js"]}
/>

## Componentes de Clase:

### Sin variable:

**Ventajas:** Más compacto y directo, especialmente cuando se trabaja con clases pequeñas y específicas.

**Uso común:** Para componentes de clase más simples.

<CodeBlock
  codeString={`
  class Saludo extends React.Component {
    render() {
        return <p>Hola, {this.props.nombre}!</p>;
    }
  }
  `} 
  language='javascript'
/>

<DynamicSandpack
dynamicFiles={{    
"/App.js": 
`import React from 'react';
import Saludo from './Saludo';

class App extends React.Component {
  render() {
    return (
      <div>
        <Saludo nombre="Usuario" />
      </div>
    );
  }
}

export default App;`, 

"/Saludo.js":
`import React from 'react';

class Saludo extends React.Component {
  render() {
    return <p>Hola, {this.props.nombre}!</p>;
  }
}

export default Saludo;`,
  }}
  
  dynamicVisibleFiles={["/App.js", "/Saludo.js"]}
/>

### Con variable:

**Ventajas:** Puede facilitar la reutilización y la referencia cuando se trabaja con componentes de clase más grandes o complejos.

**Uso común:** Para componentes de clase más complejos o cuando se desea una referencia fácilmente reutilizable.

<CodeBlock
  codeString={`
  const Saludo = class extends React.Component {
    render() {
        return <p>Hola, {this.props.nombre}!</p>;
    }
  };
  `} 
  language='javascript'
/>
