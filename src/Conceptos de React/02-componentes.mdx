# Componentes

Los componentes en React son bloques de construcción reutilizables que permiten estructurar la interfaz de usuario de una aplicación de manera modular. 
Cada componente puede contener su propio estado y lógica, y puede ser compuesto por otros componentes para construir interfaces más complejas.

### Definición de Componente:

En React un componente puede ser una función o una clase de JavaScript que devuelve elementos de React (usualmente, representados mediante JSX) que describen cómo se debe 
renderizar la interfaz de usuario.

## Componentes Funcionales:

### Sin variable:

**Ventajas:** Más conciso y a menudo se prefiere para componentes simples.

**Uso común:** Para componentes pequeños y funcionales que no necesitan ser referenciados en múltiples lugares.

<CodeBlock
  codeString={`
  function Saludo(props) {
    return <p>Hola, {props.nombre}!</p>;
  }
  `} 
  language='javascript'
/>

### Con variable:

**Ventajas:** Puede ser útil para componentes más grandes o cuando se necesita reutilizar el componente en varios lugares.

**Uso común:** Para componentes más complejos o cuando se desea una referencia fácilmente reutilizable.

<CodeBlock
  codeString={`
  const Saludo = function(props) {
    return <p>Hola, {props.nombre}!</p>;
  };
  `} 
  language='javascript'
/>

## Componentes de Clase:

### Sin variable:

**Ventajas:** Más compacto y directo, especialmente cuando se trabaja con clases pequeñas y específicas.

**Uso común:** Para componentes de clase más simples.

<CodeBlock
  codeString={`
  class Saludo extends React.Component {
    render() {
        return <p>Hola, {this.props.nombre}!</p>;
    }
  }
  `} 
  language='javascript'
/>

### Con variable:

**Ventajas:** Puede facilitar la reutilización y la referencia cuando se trabaja con componentes de clase más grandes o complejos.

**Uso común:** Para componentes de clase más complejos o cuando se desea una referencia fácilmente reutilizable.

<CodeBlock
  codeString={`
  const Saludo = class extends React.Component {
    render() {
        return <p>Hola, {this.props.nombre}!</p>;
    }
  };
  `} 
  language='javascript'
/>

### JSX (JavaScript XML)

JSX es una extensión de la sintaxis de JavaScript que se parece al XML o HTML pero se utiliza con React para describir cómo debería verse la interfaz de usuario. 
JSX facilita la creación de elementos de React de manera más concisa y legible.

<CodeBlock
  codeString={`
  const element = <h1>Hola, mundo!</h1>;
  `} 
  language='javascript'
/>

###  Unidireccionalidad de Datos

En React los datos fluyen en una dirección específica de arriba hacia abajo a través de los componentes. Esto se conoce como unidireccionalidad de datos o "flujo unidireccional". 
La idea es que el componente principal pasa datos a los componentes secundarios a través de las props.

<CodeBlock
  codeString={`
  import React, { useState } from 'react';

// Componente Padre
const App = () => {
  const [usuario, setUsuario] = useState('Usuario');

  return <MiComponente nombre={usuario} />;
};

// Componente Hijo
const MiComponente = (props) => {
  return <p>Hola, {props.nombre}!</p>;
};

export default App;
  `} 
  language='javascript'
/>

Esto ayuda a mantener la aplicación predecible y facilita la depuración ya que siempre sabes de dónde provienen los datos.
