# Iteradores

<a id="seccion1"></a>
- **For Loop:** Un bucle for se utiliza para repetir un bloque de código un número específico de veces.

<CodeBlock
  codeString={`
  for (let i = 0; i < 5; i++) {
    console.log(i); // Imprime 0, 1, 2, 3, 4
  }
  `} 
  language='javascript'
/>

<a id="seccion2"></a>
- **break y continue en un for loop:** break se utiliza para salir de un bucle antes de que se complete y continue se utiliza para saltar la iteración actual y pasar a la siguiente.

<CodeBlock
  codeString={`
  for (let i = 0; i < 5; i++) {
    if (i === 2) {
      break; // Sale del bucle cuando i es igual a 2
    }
    console.log(i); // Imprime 0, 1
  }

  for (let i = 0; i < 5; i++) {
    if (i === 2) {
      continue; // Salta la iteración cuando i es igual a 2
    }
    console.log(i); // Imprime 0, 1, 3, 4
  }
  `} 
  language='javascript'
/>

<a id="seccion3"></a>
- **While Loop:** Un bucle while se utiliza para repetir un bloque de código mientras una condición sea verdadera.

<CodeBlock
  codeString={`
  let i = 0;
  while (i < 5) {
    console.log(i); // Imprime 0, 1, 2, 3, 4
    i++;
  }
  `} 
  language='javascript'
/>

<a id="seccion4"></a>
- **Do While Loop:** Similar al bucle while pero garantiza que el bloque de código se ejecute al menos una vez incluso si la condición es falsa.

<CodeBlock
  codeString={`
  let i = 0;
  do {
    console.log(i); // Imprime 0
    i++;
  } while (i < 0);
  `} 
  language='javascript'
/>

<a id="seccion5"></a>
- **.forEach y .map:** forEach y map son métodos de iteración para arrays en JavaScript.

<CodeBlock
  codeString={`
  // forEach: Ejecuta una función para cada elemento del array
  const array = [1, 2, 3];
  array.forEach(element => console.log(element * 2)); // Imprime 2, 4, 6

  // map: Crea un nuevo array con los resultados de llamar a una función para cada elemento del array
  const doubledArray = array.map(element => element * 2);
  console.log(doubledArray); // Imprime [2, 4, 6]
  `} 
  language='javascript'
/>

<a id="seccion6"></a>
- **for ...of:** El bucle for...of se utiliza para recorrer los elementos de un iterable (como un array o una cadena).

<CodeBlock
  codeString={`
  const array = [1, 2, 3];
  for (const element of array) {
    console.log(element); // Imprime 1, 2, 3
  }
  `} 
  language='javascript'
/>

<a id="seccion7"></a>
- **for ...in:** El bucle for...in se utiliza para recorrer las propiedades de un objeto.

<CodeBlock
  codeString={`
  const obj = { a: 1, b: 2, c: 3 };
  for (const key in obj) {
    console.log(key, obj[key]); // Imprime a 1, b 2, c 3
  }
  `} 
  language='javascript'
/>

# Tips

#### Evitar modificar el array mientras se itera

Modificar un array mientras se está iterando puede conducir a resultados inesperados o errores. 
Si necesitas modificar el array considera hacer una copia antes de la iteración

<CodeBlock
  codeString={`
  const array = [1, 2, 3];

  // Incorrecto: Modificar el array mientras se itera
  for (const value of array) {
    array.push(value * 2); // Esto puede causar un bucle infinito
  }

  // Correcto: Hacer una copia antes de la iteración
  const copyArray = [...array];
  for (const value of copyArray) {
    array.push(value * 2);
  }
  `} 
  language='javascript'
/>

# Tips

#### Manejo de Eventos

Asegurarse de asignar funciones de manejo de eventos de manera correcta.

<CodeBlock
  codeString={`
  // Error
  document.getElementById('miBoton').addEventListener('click', miFuncion());

  // Correcto
  document.getElementById('miBoton').addEventListener('click', miFuncion);
  `} 
  language='javascript'
/>

#### Referencias a Elementos del DOM

Garantizar que el DOM esté completamente cargado antes de interactuar con él.

<CodeBlock
  codeString={`
  // Error
  document.addEventListener('DOMContentLoaded', function () {
    // Intentar acceder a elementos del DOM aquí
  });

  // Correcto
  document.addEventListener('DOMContentLoaded', function () {
    // Acceder a elementos del DOM aquí
  });
  `} 
  language='javascript'
/>

#### this en Funciones de Manejo de Eventos

Asegurarse de que el valor de **this** sea el esperado en funciones de manejo de eventos.

<CodeBlock
  codeString={`
  // Error
  document.getElementById('miBoton').addEventListener('click', function () {
    // 'this' no se refiere al elemento del botón
  });

  // Correcto
  document.getElementById('miBoton').addEventListener('click', () => {
    // 'this' se refiere al elemento del botón
  });
  `} 
  language='javascript'
/>
