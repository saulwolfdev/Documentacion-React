# Programación Funcional:

La programación funcional es un paradigma de programación que trata el cálculo como evaluación de funciones matemáticas y evita cambiar estados y mutabilidad. 
En JavaScript moderno, la programación funcional se ha vuelto más prominente gracias a la adopción de características como funciones de orden superior, funciones flecha, y 
métodos de array de alto orden.

<a id="seccion1"></a>
### Cuándo usar programación funcional en JavaScript:

**Operaciones Inmutables y Sin Efectos Secundarios:**
Cuando necesitas realizar operaciones sin modificar los datos existentes y sin efectos secundarios.

**Manejo De Datos:**
- Para procesar y transformar datos de manera declarativa.
- Cuando trabajas con grandes conjuntos de datos y quieres evitar efectos colaterales.

**Manejo de Eventos y Callbacks:**
En situaciones donde se utilizan funciones de orden superior para manejar eventos y callbacks, como en aplicaciones basadas en eventos.

**Concurrente y Paralelo:**
En situaciones donde la programación funcional facilita la escritura de código concurrente y paralelo, ya que las funciones puras son más fáciles de razonar en entornos 
concurrentes.

<a id="seccion2"></a>
### Funciones de Orden Superior (Higher-Order Functions):

Estas son funciones que pueden tomar otras funciones como argumentos o devolverlas como resultado.

<CodeBlock
  codeString={`
  const multiplicarPorDos = x => x * 2;
  const aplicarOperacion = (numero, operacion) => operacion(numero);

  console.log(aplicarOperacion(5, multiplicarPorDos)); // Resultado: 10
  `} 
  language='javascript'
/>

<a id="seccion3"></a>
### Funciones Puras: 

Son funciones que producen el mismo resultado dado el mismo conjunto de entradas y no tienen efectos secundarios esto facilita el razonamiento y la comprensión del código.

<CodeBlock
  codeString={`
  // Función impura
  let contador = 0;
  const incrementarContador = ( ) => {
    contador++;
    return contador;
  };

  // Función pura
  const sumar = (a, b) => a + b;
  `} 
  language='javascript'
/>

<a id="seccion4"></a>
### Inmutabilidad:

Los datos en la programación funcional son inmutables es decir no pueden ser modificados una vez creados. En lugar de modificar los datos existentes, se crean nuevos datos.

<CodeBlock
  codeString={`
  // Modificación no funcional
  let arr = [1, 2, 3];
  arr.push(4); // Modifica el array existente

  // Enfoque funcional
  let arr2 = [1, 2, 3];
  let newArr = [...arr2, 4]; // Crea un nuevo array
  `} 
  language='javascript'
/>

<a id="seccion5"></a>
### Métodos de Array de Alto Orden:

Métodos como **map**, **filter**, y **reduce** son fundamentales en la programación funcional en JavaScript. Estos métodos operan en arrays y permiten realizar operaciones de 
manera más declarativa.

<CodeBlock
  codeString={`
  const numeros = [1, 2, 3, 4, 5];

  // Map: Transforma cada elemento del array
  const duplicar = numeros.map(numero => numero * 2);

  // Filter: Filtra elementos según una condición
  const pares = numeros.filter(numero => numero % 2 === 0);

  // Reduce: Combina los elementos en un solo resultado
  const suma = numeros.reduce((total, numero) => total + numero, 0);
  `} 
  language='javascript'
/>

### Conclusion:

La programación funcional puede hacer que el código sea más fácil de entender, mantener y razonar, especialmente en aplicaciones grandes y complejas. 
Sin embargo, la elección de utilizar programación funcional depende del contexto y de los requisitos específicos del proyecto.
